# Changelog

All notable changes to Deck Shelves will be documented in this file.

The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.1.0/),
and this project adheres to [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

## [Unreleased]

### Added

- **Peripheral context signals** ([`peripheralsState.ts`](src/runtime/peripheralsState.ts), [`peripherals.py`](peripherals.py), [`deviceState.ts`](src/runtime/deviceState.ts)). A new **Peripherals** category of visibility / trigger conditions: **a controller is connected** (any controller beyond the built-in one), **headphones are connected** (the default audio output is a headset) and **a specific Bluetooth device is connected** (chosen from your paired devices by name). All three invert ("no controller connected", and so on). The Bluetooth and headphone readings hit the backend only when a shelf that uses them resolves — no polling — and stay inert when the platform can't report them.
- **Cross-platform device signals.** CPU, memory and external-display readings now work on **Windows and macOS** as well as SteamOS / Linux ([`perf_probe.py`](perf_probe.py), [`display_state.py`](display_state.py)): they use `psutil` when it is installed and fall back to per-OS system calls otherwise, and each stays fail-soft, so a signal a platform can't provide simply leaves its rule inert.
- **System information + bug report name the real OS on every platform** ([`host_os.py`](host_os.py), [`diagnosticsInfo.ts`](src/runtime/diagnosticsInfo.ts)). A backend host-identity probe (Python `platform` + `/etc/os-release`) reports SteamOS, the specific Linux distro (Bazzite / HoloISO / ChimeraOS / other), Windows or macOS with version and architecture — so System information and the pre-filled bug report are accurate off the Steam Deck, mapping to the correct issue-form OS option instead of defaulting to "SteamOS".
- **Online store cookie reading is now cross-OS** ([`main.py`](main.py) `_decrypt_chromium_cookie`). The Chromium cookie decryption is OS-dispatched: Linux (as before), macOS via the Keychain-derived key, and Windows via native DPAPI + AES-256-GCM (CNG through `ctypes`, no added dependency). Fail-soft everywhere — any failure returns nothing so region / wishlist authentication falls back to the public path instead of breaking.
- **Discoverable built-in catalogues in the public API** ([`@deck-shelves/api`](api/src/types.ts)). Integrations can enumerate what the plugin ships: `listTriggerCatalog()` (every visibility / trigger kind with its category and whether it inverts), `listShelfTemplates()` (the built-in shelf templates) and `listShortcuts()` (the gamepad shortcuts with their default and current combos). See [docs/plugin-api.md](docs/plugin-api.md).
- **One-tap bug report from the About page.** The **Report an issue** button now gathers your diagnostics (runtime, system, co-loaded plugins and a configuration summary) plus the recent developer logs and opens a pre-filled issue that follows the bug-report template — title, environment fields and context already filled in ([`issueReport.ts`](src/core/issueReport.ts)).
- **Visibility Rules v2 — a predicate tree for shelf visibility.** A `visibility: { mode: 'any' | 'all', rules[] }` tree on regular shelves, smart shelves and saved filters ([`smartShelves.ts`](src/steam/smartShelves.ts) `evalVisibility` / `evalVisibilityRules` / `nextVisibilityFlip`) generalises the legacy `visibleHours` / `visibleDaysOfWeek` — which keep working unchanged via `isInVisibilityWindow` delegation — into an **any/all** combination of rule kinds: `timeWindow`, `dayOfWeek`, and the device kinds `battery` (below %), `charging`, `offline`, `externalDisplay`, `resolution` (min width) and `ultrawide`. A new Visibility editor ([`VisibilityRulesEditor.tsx`](src/components/qam/modals/editShelf/VisibilityRulesEditor.tsx)) adds/removes rows, toggles any/all and offers Evenings / Weekends presets. The schema is passthrough and the backend sanitizer keeps unknown scalar params, so it is forward-compatible. Clock-based rules re-arm a **one-shot** timer at the next boundary (no polling); device rules re-evaluate on hardware events.
- **Device-state context signals** ([`deviceState.ts`](src/runtime/deviceState.ts) + [`display_state.py`](display_state.py)). A live, event-driven, **cached** device snapshot — battery level / charging (battery subscription), offline (settings), external-display / docked (backend DRM connector read of `/sys/class/drm/*/status` via the `get_display_state` RPC) and screen resolution / ultrawide (the Big-Picture window's real dimensions; the SharedJSContext window reports 1). Refreshes **only** on Steam `DisplayManager` state changes (debounced + change-gated) and battery notifications (gated to meaningful changes) — no polling, no timers. Cross-platform: external-display is fail-soft off SteamOS (`supported: false` → the rule is inert), while resolution / ultrawide read the display on any OS.
- **Profile auto-switch triggers.** Each saved profile can carry a `trigger` (a Visibility tree); with the master **auto-switch** toggle on — surfaced in the QAM, the sidecar and Settings → Profiles — the first profile whose trigger matches is applied automatically. The resolver runs in a **boot-installed background module** ([`runtime/profileTriggers.ts`](src/runtime/profileTriggers.ts) + [`resolveTriggeredProfile`](src/steam/smartShelves.ts)) so it fires on the home regardless of whether the settings panel is open; it is transition-based (applies only on a change, never mid-edit, and can't loop) and event/clock-driven with no polling. A per-profile trigger editor ([`SetProfileTriggerModal.tsx`](src/components/qam/modals/SetProfileTriggerModal.tsx)) reuses the visibility editor; triggers round-trip through export / import and are summarised in System information.
- **Optional shelf-linking for profiles.** Saving a profile now offers a **Link shelves to profile** toggle (in the QAM save dialog and Settings → Profiles). Linked profiles swap the displayed shelves when you switch to them; unlinked profiles apply every other setting but leave your current shelves in place ([`profiles.ts`](src/features/settings/controller/profiles.ts) — `linkShelves`, round-tripped by the sanitizer).
- **More context-signal visibility kinds.** The Visibility Rules v2 tree (and profile triggers) gained **time-context** kinds — `weekend`, `timeOfDayPeriod` (morning / afternoon / evening / night), `season` (hemisphere-aware) and `holiday` (user-editable MM-DD ranges) — all pure date math ([`domain/timeContext.ts`](src/domain/timeContext.ts)); **session-context** kinds — `lastGameSource` (Steam vs non-Steam of the last-played game) and `gameRunning` — event-driven via app-lifetime notifications ([`runtime/sessionState.ts`](src/runtime/sessionState.ts)); and **performance-context** kinds — `highCpu` and `lowMemory`, read on demand from `/proc` with **no background timer** (only fetched when a shelf that uses them resolves, short-cached) ([`perf_probe.py`](perf_probe.py) + [`runtime/perfState.ts`](src/runtime/perfState.ts)).
- **Two new usage filter types** — **`recentlyActive`** (played in the last two weeks, via Steam's rolling 2-week playtime) and **`neglected`** (played before but not in the last N days; never-played titles excluded) ([`steam/index.ts`](src/steam/index.ts)).
- **Auto-pin shelves to the top on a condition.** A shelf (regular or smart) can carry an `autoPin` predicate — the same rule tree as visibility — and floats to the top of the home while it matches, dropping back when it clears ([`domain/shelfOrder.ts`](src/domain/shelfOrder.ts) `applyAutoPin` + [`HomeInject.tsx`](src/components/HomeInject.tsx)). Opt-in and a stable partition, so a home with no auto-pinned shelves is byte-for-byte unchanged.
- **Auto-collapse shelves on a condition.** With the global Auto-collapse toggle on, a shelf can carry an `autoCollapse` predicate (same rule tree) and/or a **collapse-when-empty** flag; it renders folded to just its header while it matches ([`DeckRow.tsx`](src/components/DeckRow.tsx), threaded from `HomeInject`). Opt-in per shelf; off by default.
- **Auto-switch notification + a notifications toggle.** When a profile trigger auto-applies a profile, a Steam toast shows which profile was switched to (the user-visible indication for context-driven switching). A **Disable notifications** behaviour toggle suppresses the plugin's own toasts only ([`runtime/notify.ts`](src/runtime/notify.ts) + [`runtime/profileTriggers.ts`](src/runtime/profileTriggers.ts)).
- **Snapshot recovery from the crash screen.** If the settings UI ever fails to render and you have settings snapshots, the error screen now offers **Restore a snapshot** → a picker that restores the chosen backup ([`SnapshotRecovery.tsx`](src/components/SnapshotRecovery.tsx) + [`RestoreSnapshotModal.tsx`](src/components/qam/modals/RestoreSnapshotModal.tsx)); a QA harness (`qa:qam-error` + `qa:snapshot-recovery` runtime flags) previews the whole flow.
- **Live context readout in the dev overlay** — resolution, docked, battery, CPU / memory and last-played source appear in the on-home debug overlay ([`DebugOverlay.tsx`](src/components/DebugOverlay.tsx)).
- **Three new filter types** ([`steam/index.ts`](src/steam/index.ts), [`customfilters.ts`](src/domain/customfilters.ts), [`FilterItemOptions.tsx`](src/components/filter/FilterItemOptions.tsx)). **System compatibility** (keep games available on the current platform), **Remote Play location** (`local` / `remote` / `remote-only` / `both`, from `per_client_data`) and **Price range** (min/max in your store currency, gated on online features).
- **Plugin version shown on every settings page and in the QAM** ([`VersionFooter.tsx`](src/components/ui/VersionFooter.tsx)). A standardised version footer on the About / Settings pages and the Quick Access panel, matching the About page and System information — so the version the bug-report form asks for is always visible.
- **CSS Loader: real active theme names in System information** ([`css_themes.py`](css_themes.py) + [`cssLoaderDetect.ts`](src/core/cssLoaderDetect.ts)). Reads `~/homebrew/themes/*/{theme.json,config_USER.json}` off disk (via the `get_css_loader_themes` RPC) to list the actual active theme name(s) and installed count instead of a structural heuristic. Gated on CSS Loader being installed, fail-soft, and `DECKY_HOME`-based so it also works with the Windows CSS Loader beta.
- **A guided feature tour.** A first-run showcase walks through the main features — custom shelves, smart shelves, filters, personalization, quick search, side navigation, profiles & triggers, online features and advanced tools — one step at a time ([`ShowcaseModal.tsx`](src/components/qam/modals/ShowcaseModal.tsx)). It appears once (tracked by `showcaseSeen`) and can be replayed anytime from the About page's **Replay feature tour** button.
- **Inverse triggers.** Any invertible condition can now be used negated — "stopped charging", "went online", "no game running", "no external display", "not ultrawide" — carried as a `not` flag on the rule ([`evalVisibilityRule`](src/steam/smartShelves.ts)). The editor never lets a condition and its inverse coexist.
- **A trigger on the Default profile.** The synthetic Default profile can carry its own auto-switch trigger (stored in `factoryProfileTrigger`), applied — non-destructively, keeping your shelves — when it matches ([`profileTriggers.ts`](src/runtime/profileTriggers.ts)). The trigger editor also opens from the Default profile row.
- **Community links on the About page** — quick buttons for Discord and Reddit next to the existing GitHub / report-issue links ([`SupportPage.tsx`](src/components/about/SupportPage.tsx)).
- **Per-area notification toggles.** Beneath the master "disable notifications" toggle, sub-toggles let you silence just Shelves, Profiles, Filters or Triggers notifications while keeping the rest — all on by default, shown only while notifications are enabled ([`NotificationAreaToggles.tsx`](src/components/qam/NotificationAreaToggles.tsx), `notificationsDisabledAreas`).

### Changed

- **The "Default" profile no longer disables the plugin or clears your shelves.** Applying it resets settings to defaults but keeps the plugin **enabled** and, unless you tick the new **Also reset shelves** option in its confirm dialog (off by default), leaves your current shelves untouched ([`profiles.ts`](src/features/settings/controller/profiles.ts) `applyFactoryProfile`). "Reset everything" remains the full wipe.
- **Auto-pin / auto-collapse are available on every shelf.** The auto-pin and auto-collapse rule editors (previously smart-shelf only) now appear on regular shelves too, under the shelf editor's **Display** tab; the global **Auto-collapse** and **Disable notifications** toggles moved to the QAM **Behaviour** section ([`EditShelfModal.tsx`](src/components/qam/modals/EditShelfModal.tsx), [`DeckQAMSettings.tsx`](src/components/DeckQAMSettings.tsx)).
- **Online store queries use your store country.** The featured / specials and price fetches were hardcoded to `cc=us`; they now use the detected store country code (store cookie / current user, falling back to `us`), so prices and featured content match your region ([`onlineStore.ts`](src/core/onlineStore.ts)).
- **Auto-switched profiles return to their previous state.** When a trigger stops matching, the profile reverts to whatever settings were active before it fired — captured in memory as a temporary override — instead of staying applied ([`profileTriggers.ts`](src/runtime/profileTriggers.ts)). The switch and the revert only notify when the profile actually changes, and the revert toast names the profile that was deactivated.
- **The trigger / visibility editor is grouped and iconified.** Conditions are now organised into collapsible sections by type — Time & date, Session, Power, Connectivity, Display, Performance — each condition with its own icon; invertible conditions render as a paired "one or the other" control ([`VisibilityRulesEditor.tsx`](src/components/qam/modals/editShelf/VisibilityRulesEditor.tsx)). The trigger menu / button only appears while auto-switch is enabled.
- **Every confirmation dialog accepts the Menu button to confirm**, matching the shelf editors — a project-wide standard applied to the profile-trigger, rename, save, delete and generic confirm dialogs.
- **All notifications share one look** — the branded logo plus a small per-type icon ([`notify.ts`](src/runtime/notify.ts) / [`notify.tsx`](src/components/notify.tsx)), so profile switches and other plugin toasts read consistently.
- **Leaner snapshot retention.** Automatic snapshots now age out after 7 days, and the backups folder is capped at **10 snapshots total** (auto + manual + imported) — over the cap the oldest are pruned, **auto snapshots first**, so your manual and imported backups are kept longest ([`storage.py`](storage.py)). Import / export are unaffected.
- **Create / save / delete / duplicate / rename actions now confirm with a notification** (shelves, smart shelves, profiles and saved filters), honouring the master and per-area notification toggles.
- **Trigger / visibility editor polish:** pick a condition's inverse directly and it swaps in place (no remove-and-re-add); CPU, memory, frame-rate and last-played-source conditions gained inverses; and resolution options read as friendly labels (Full HD / QHD / 4K …).

### Fixed

- **The image picker opens in your real Pictures folder on any OS.** It defaulted to a hardcoded `/home/deck/Pictures`; it now resolves `~/Pictures` through the backend (cross-OS, any account) with a home-directory fallback ([`imagePicker.ts`](src/core/imagePicker.ts), [`userPaths.ts`](src/core/userPaths.ts)).
- **The Quick Access sidecar no longer closes by itself.** It used to collapse whenever its window received a focus event — which Steam raises spuriously (a notification dismissing, a controller or dock event) while the panel was still open. It now closes only when the Quick Access menu was genuinely hidden first ([`DeckQAMSettings.tsx`](src/components/DeckQAMSettings.tsx)).
- **A single malformed field can no longer reset your settings.** A settings load that failed validation could fall back to defaults and overwrite your saved configuration; the loader now preserves your existing settings and merges only what it can parse ([`settingsStore.ts`](src/store/settingsStore.ts)).
- **The first shelf no longer glues to the very top when native recents are hidden.** With hide-recents on and a hero background, the promoted first shelf was pulled up as a whole box (`margin-top: -56px`), sticking its cards and title to the screen top. It now bleeds only the hero *art* under the transparent header (like the non-promoted first shelf already did), so the cards/title keep the same top spacing the native recents row had ([`shelfStylesheetTemplate.ts`](src/components/shelf/shelfStylesheetTemplate.ts)).
- **The price-range filter now actually applies to unowned store games.** A store / wishlist shelf is meant to surface games you *don't* own so you can buy them — but the price child filter waved through **every** game not in your local library (`!app → true`), so free-to-play and out-of-range titles slipped past the min / max bound. Price kinds now evaluate every store game against the price cache by app id (fetching prices first, like the discount filter already did): an unowned game appears **only when its real price is in range**, and free / unpriced titles are excluded ([`steam/index.ts`](src/steam/index.ts) `evaluateWishlistChildItem` / `applyWishlistChildFilter`).
- **Recurring ~2 s UI freeze every 30 s on the home (dropped controller input / "stuck" navigation).** The price cache was re-parsed from `localStorage` once per app during shelf resolution (`readPriceCache` / `readPriceCacheEntry`), so a home with 100+ cards and a populated cache blocked the main thread ~2 s on every refresh — including the 30 s fallback poll — which swallowed gamepad presses and made focus appear stuck on a shelf. The parse is now memoised, keyed on the raw cache string so it self-invalidates on any write, so one resolution pass parses once ([`onlineStore.ts`](src/core/onlineStore.ts), [`steam/index.ts`](src/steam/index.ts)). Diagnosed on-device: a CDP heartbeat showed 2.2 s main-thread stalls at exact 30 s intervals and a CPU profile pinned `readPriceCache` / `evalDiscount` as the hot frames; after the fix, 0 stalls (max 183 ms) and those frames dropped out of the profile.
- **The profiles "apply / delete" confirmation now grabs gamepad focus.** The Profiles settings page used a hand-rolled fixed-overlay dialog that never received focus, so applying the "Default" profile was too easy to trigger blindly. It now uses the standard Decky `ConfirmModal` (via `confirmAction`) ([`ProfilesDetail.tsx`](src/components/settings/details/ProfilesDetail.tsx)).

## [3.0.2] - 2026-07-10

### Added

- **Snapshots — versioned settings history** (Settings → Advanced). Rolling backups of the settings file at `settings/deck-shelves/backups/settings-<ts>.json` ([`storage.py`](storage.py) + [`SnapshotsSection.tsx`](src/components/settings/details/SnapshotsSection.tsx)). Automatic snapshots are throttled to one per 24 h (`AUTO_BACKUP_CAP` = 12, oldest pruned); user-created (`manual`) and `import` snapshots carry an origin icon. Per-row restore / export / delete plus delete-all; restore is itself undoable. Backend RPCs `list_backups` / `restore_backup` / `create_backup` / `export_backup` / `import_backup` are path-traversal guarded. Renamed from "Backups" to avoid clashing with the file export/import "Backup" area.
- **Cache management** (Settings → Advanced). Per-key size readout + Clear (empty groups disabled) for the persisted localStorage caches, plus a confirmed clear-all ([`cacheRegistry.ts`](src/runtime/cacheRegistry.ts)); the smart-shelf memo cache folds into clear-all rather than its own row.
- **System information panel** (Settings → Advanced, formerly "Diagnostics"). Read-only runtime detection — version, generic OS + Steam version, active theme (heuristic + CSS-Loader style count) and co-loaded plugins (TabMaster / UnifiDeck / Non-Steam Badges) — plus a **summary of the active Deck Shelves configuration** (counts, core toggles, the Additional-features section, and the Global-visual toggles + values). Copy button and individually gamepad-focusable rows ([`diagnosticsInfo.ts`](src/runtime/diagnosticsInfo.ts) + [`DiagnosticsSection.tsx`](src/components/settings/details/DiagnosticsSection.tsx)).
- **Developer mode + Developer tab.** A `devModeEnabled` toggle at the end of Advanced reveals a Developer tab ([`DeveloperDetail.tsx`](src/components/settings/details/DeveloperDetail.tsx)) hosting the on-home debug-overlay controls, the source-resolver inspector, the reactive focus chain and the diagnostic log viewer.
- **On-home debug overlay** ([`DebugOverlay.tsx`](src/components/DebugOverlay.tsx)). fps / frame ms, shelf / node / focusable counts, per-shelf node counts, a reactive focus readout (ancestor path + the focused element's full CSS class list + the `ds-*` classes applied along the focus chain), and weight-coloured render outlines swept across every Steam document. Live-configurable corner / orientation / transparency / content. Mounted only when enabled (zero overhead off), `pointerEvents:none`, rAF cleaned up on unmount. Gated by Developer mode (or the legacy `ds-debug` / `?debug=1` escape hatch).
- **Source-resolver inspector** ([`SourceResolverInspector.tsx`](src/components/settings/details/SourceResolverInspector.tsx)). Live-evaluates a shelf and lists the resolved app ids per filter step.
- **Adjustable description text size** — a global + per-shelf `descriptionScale` slider (100–200 %), replacing the old description-large boolean.
- **Statistics → Library trends.** A KPI/numbers row plus additional, more varied chart types in the Statistics tab ([`StatisticsDetail.tsx`](src/components/settings/details/StatisticsDetail.tsx)).

### Changed

- **Standardised iconography across Settings and the QAM.** Export uses an up-arrow, import a down-arrow, delete a trash can and restore its own icon (replacing the floppy); collapsible sections carry icons in every settings tab, and the QAM import/export icons match the config pages.

### Fixed

- **Full-page hero shelves bobbed ~6px vertically per card during lateral navigation.** `DeckRow`'s `handleFocusedCard` re-centered the shelf on *every* card focus via three tolerance-free scroll passes (`scrollIntoView` + local-ancestor + Steam-BP-viewport fallbacks) — unlike `maybeCenter`, which has a 32px tolerance — so moving card-to-card kept re-scrolling the whole shelf (hero art included) by ~6px, alternating per card. It now skips the vertical re-center on lateral moves within the same shelf (previous card tracked via `__ds_prev_centered_card`), re-centering only when focus enters the shelf from another row ([`DeckRow.tsx`](src/components/DeckRow.tsx)). Diagnosed on-device via a time-series CDP capture: the hero was rock-stable relative to its cards (`heroVsRow` constant) while the whole shelf oscillated 0↔−6px in the viewport.
- **Update checks + online features were blocked by a failing connectivity probe.** `isOnline()` probed `store.steampowered.com/favicon.ico` in default CORS mode, so the cross-origin fetch REJECTED with "Failed to fetch" (making the intended opaque-response branch dead code) — which pinned `isOnline()` to `false` and short-circuited `checkForUpdate` (no update banner, even on the beta channel) and every online-gated feature. Added `mode: "no-cors"` so a reachable host resolves as an opaque response instead of throwing ([`connectivity.ts`](src/core/connectivity.ts)). Diagnosed on-device via CDP (GitHub API fetch succeeded; the Steam probe returned `TypeError: Failed to fetch`, then `type: "opaque"` with `no-cors`).
- **Quick Search: the on-screen keyboard now closes when a game is found.** On a match the focused text input is blurred across every Steam document and the keyboard is dismissed via all known close methods ([`SearchOverlay.tsx`](src/features/search/SearchOverlay.tsx)).
- **Card decorations no longer float over the search keyboard / Side Nav.** The portaled "new/discount" badge and the friends-playing avatars now hide while an ambient overlay (Quick Search / Side Nav) is open — the overlay-active detector treats a present `.ds-search-overlay` / `.ds-sidenav-overlay` as covering the home ([`overlayState.ts`](src/components/shelf/overlayState.ts), [`BadgeFocusOverlay.tsx`](src/components/shelf/BadgeFocusOverlay.tsx), [`FriendsAvatarOverlay.tsx`](src/components/shelf/FriendsAvatarOverlay.tsx)).
- **Unit tests resolve the workspace packages from source.** [`vitest.config.ts`](vitest.config.ts) now aliases `@deck-shelves/host` / `@deck-shelves/api` to their `src` (mirroring the `tsconfig` paths and the build config), so CI — which runs the tests before any package is built to `dist` — no longer fails with `ERR_MODULE_NOT_FOUND`.
- **Coexistence checks stop flagging false positives.** The UnifiDeck check no longer treats importing our own guarded `integrations/registry` helper as a hard dependency, and the TabMaster try/catch-guard scan now excludes test files (mocks aren't runtime access) ([`checks/plugins/tabmaster.sh`](checks/plugins/tabmaster.sh), [`diagnosticsInfo.ts`](src/runtime/diagnosticsInfo.ts)).

## [3.0.1] - 2026-07-05

### Added

- **Local dev deploy without SSH** ([`scripts/deploy/deploy-local.mjs`](scripts/deploy/deploy-local.mjs)). `pnpm run deploy:local` / `deploy:local:hard` build and install the plugin into **this machine's** Decky Loader plugin dir (default `~/homebrew/plugins`, overridable via `DECKY_PLUGINS_DIR` / `DECKY_HOME`) — for developing directly on a Deck / Linux / Windows box that already has Decky, with no remote SSH. Cross-OS Node; it never installs Decky (clear error + override hint when the homebrew dir is absent). `:hard` best-effort restarts `plugin_loader.service` + Steam on Linux. Also documented the opt-in `DECK_CDP_HOST=127.0.0.1` + `DECK_CDP_PORT=8080` for pointing `deckprobe` at local Steam (shipped default `8081` unchanged, so Deck/CI flows are unaffected).
- **Portable snapshot export/import** ([`domain/snapshot.ts`](src/domain/snapshot.ts) + controller `exportSnapshot` / `importSnapshot`). Bundles regular shelves, smart shelves, saved filters and saved smart filters into a single versioned JSON; `merge` appends with id de-duplication, `replace` overwrites. Format-compatible with the existing per-concept exports.
- **Plugin-to-plugin export/import handlers** ([`pluginApi.ts`](src/core/pluginApi.ts) + [`api/src/types.ts`](api/src/types.ts)). New `registerExportHandler` / `registerImportHandler` (+ `getRegisteredExportHandlers` / `getRegisteredImportHandlers`) let a third-party plugin offer "Export to format X" / "Import from format Y". Handlers are format-agnostic — they translate between the snapshot JSON and their own format, so the transfer is lossless. Bridged by controller `exportViaHandler` / `importViaHandler` and surfaced in Settings → Backup. Additive to the frozen contract (no `version` bump).

### Changed

- **QAM Profiles section reworked into a shelves-style list** ([`ProfilesSection.tsx`](src/components/qam/sections/ProfilesSection.tsx)). A single CSS-Loader-style dropdown selects a profile — with **"Save current" and "Reset to default" as dropdown entries** (no separate buttons) — and the saved profiles are a **reorderable list** below it (drag / up-down; the order also drives the dropdown). Each row mirrors the shelves list: an **active-profile check**, an **eye** (crossed when hidden), the left-aligned name, and a `[…]` context menu (apply / update / duplicate / rename / hide-show / export / delete). **Hidden profiles** (`profiles[].hidden`, persisted) stay in the list but are omitted from the quick-select dropdown. New controller actions `toggleProfileHidden` / `setProfilesOrder` + a small `RenameProfileModal`. In the sidecar the profile selector and the list each get their own eye (`profileDropdown` / `profileList`) via a shared `HideableRow`, and the list collapses to a single "Profiles list" item (new `profile_list_label` string, all 19 locales). The Settings-page profiles panel is unchanged.
- **Default logo↔info gap is now 10px (was 8px).** The description-to-logo spacing (`descriptionLogoGap`) defaults to `10` for new shelves and for any shelf that never set the value explicitly ([`DeckRow.tsx`](src/components/DeckRow.tsx), [`Shelf.tsx`](src/components/Shelf.tsx), [`PerShelfHero.tsx`](src/components/shelf/PerShelfHero.tsx), both shelf editors). Shelves with a saved gap are unaffected.
- **"See more" card is dynamic on library shelves (#65).** The trailing "See more" tile now appears only when the shelf's source actually has more games than the configured limit (e.g. it is hidden when 18 games fill a limit-20 shelf). Manually hiding it via the per-shelf / global toggle is unchanged; online (wishlist / store) shelves still always link out. Driven by a new optional `onResolveTotal` hook on `resolveShelfAppIds` reported by the collection / filter resolvers; when the count is unknown the tile is shown, so nothing regresses.

### Fixed

### Fixed

- **Composite shelves were dropping nearly every item to the composite-level "Online filters" tab.** The discount evaluator returns `false` for any appid missing from the price cache (i.e. every offline-owned game), so applying the composite-level `childFilter` to the merged set silently filtered out owned collection / filter items along with the intended wishlist filtering. The resolver now MERGES the composite-level online filter items into each online child's own `childFilter` BEFORE resolving — wishlist / store branches handle them natively with their pre-fetched price map, while collection / tab / filter children are left untouched. Matches the per-source semantics described in the spec: each source applies its own criteria, merge, then sort.
- **Composite shelves were also dropping locally-owned items at render time when the "exclude owned" toggle was on.** `Shelf.tsx`'s `eligibleForOwnedHide` gate previously matched any item on a composite-with-online-child shelf, but the appid-based hide and the name-based dedup against `ownedNames` were dropping collection items because the user obviously owns the games they put in their own collections. New `isStoreFallback` proxy distinguishes online-origin ids (`name` is the synthetic `App {id}` because there's no local appStore overview) from offline-origin ids (real meta from the local overview). For composite shelves, only the online-origin items go through the hide check; direct online shelves keep the prior behaviour since all their items are online by definition.
- **Composite shelves rendered out-of-order whenever the parent shelf had a non-manual sort.** Each child source resolved with the same `sort`/`sortReverse` forwarded, but `mergeCompositeResults` interleaves children via round-robin (union) or filters by membership (intersection) — both discard the per-child ordering. The resolver now runs a single `applySortToIds` pass over the merged result (after the composite-level `childFilter`), with `getPriceMap` warmed up first when any sort key is a price key (`price_low` / `discount_high` / `original_price_high`). Pool is built from raw per-id `appStore.GetAppOverviewByAppID` lookups so timestamp fields the comparator needs (`rt_original_release_date` / `rt_purchased_time` / `metacritic_score`) stay present — `getAllAppOverviews()`'s fallback path strips those via `normalizeAppOverview`, which would otherwise reduce every comparator to all-ties and preserve the merge order. Skips for `manual` and `random` since those aren't ordered at the resolver layer.
- **Composite shelves with an online (wishlist / store) child didn't expose the "exclude owned" toggles in the editor, and the toggles weren't actually filtering at render-time even after the per-child propagation was added.** Two-part fix:
 1. Editor: Source tab toggles now render when any source is online (was gated only on the primary source type). Save propagates values to every online child uniformly. Initial load reads back from the first online child.
 2. Renderer: `Shelf.tsx`'s `excludeOwned` / `effectiveNonSteam` / `effectiveCloud` / `shouldHideOwned` flags were gated on `isOnlineShelf` (direct wishlist/store only), so a composite shelf's wishlist child carrying `excludeOwned: true` only filtered via the resolver's appid-based dedup — missing same-name games owned via non-Steam shortcuts (from other storefronts; no Steam appid match). The render-time name-dedup now reads the toggles from the first online child of a composite source and applies the same hide gate, so Steam wishlist items the user owns elsewhere by name disappear from the composite row too.

- **Composite shelves with an online (wishlist / store) child rendered the wishlist items with `#<appid>` placeholders and the install indicator + "Not installed" status text** instead of real names without install state. `Shelf.tsx`'s `isOnlineShelf` check only matched direct wishlist / store sources, so composite with a wishlist child wasn't triggering the Steam Store API name-fetch path AND wasn't hiding the install-state visuals. New `compositeHasOnlineChild` flag: `true` when source.type === 'composite' AND any child is wishlist / store. Adds composite to the `needsExternalNames` gate (online-API name resolution kicks in for non-owned appids in the row) AND to the `effectiveHideInstallIndicator` + `effectiveHide` (status line) gates (shelf-wide, so owned cards in the same composite also drop those visuals for visual coherence). CDP-validated: wishlist child `appid 6300` renders as "Dreamfall: The Longest Journey" with no install indicator, no "Not installed" status text.

- **Hero art occasionally flashed a broken-image glyph for a frame when the first shelf was in the recents slot** (no native hero behind to mask it). Root cause: `transition: opacity 0.06s` was symmetric, so when the fallback chain advanced to a new URL (e.g. CDN returned a corrupt 200 OK), the img faded 1 → 0 over 60 ms, leaving a window where the browser's broken-img placeholder was visible at partial opacity. Transition is now ASYMMETRIC: the 0 → 1 up-leg keeps the gentle 60 ms fade (visual continuity when a real image decodes), but 1 → 0 is instantaneous. Fallback advances are now flicker-free.

- **"Shelf as recents" (experimental) was STILL forcing alphabetical sort on shelves with `sort = "manual"`** after the initial fix. Root cause: forwarding `sort='manual'` straight to the resolver is a no-op (manual is a placeholder for "use `manualOrder` externally") — the resolver returned the composite source's raw union order, which the user read as alphabetical-ish. `recentsReplace.tsx` now mirrors `Shelf.tsx`'s two-step manual handling: resolve using the user's `manualBaseSort` + `manualBaseSortReverse` (multi-key chain + per-key asc/desc), then `applyManualOrder` re-orders the result with pinned items first. Cache key extended with `manualBaseSort` / `manualBaseSortReverse` / `manualOrder` fingerprint so edits to ANY of those re-resolve the promoted shelf immediately. Filter sources get the same `source.filter.sort` swap Shelf.tsx applies.

- **Smart shelves with the new modes (`friends_playing` / `low_battery_mode` / `almost_finished` / `couch_gaming` / `coop_ready` / `party_games` / `short_battery` / `long_session_night` / `travel_mode` / `hidden_gems` / `never_touched_classics` / `recent_hidden_installs` / `monthly_spotlight` / `seasonal_rotation`) were being silently dropped on save.** `main.py`'s `valid_modes` set was hardcoded with only the round-1 + media + early v2 modes; every newer mode failed the `ss_mode not in valid_modes` check and was filtered out during sanitisation, so the shelf vanished after every save attempt. Set now mirrors `INTERNAL_SMART_MODES` in `src/steam/smartShelves.ts`; comment in `main.py` flags the three locations that must stay in sync when adding future templates.

- **"Shelf as recents" (experimental) forced alphabetical sort regardless of the shelf's configured ordering.** `recentsReplace.tsx`'s `scheduleResolve` was calling `resolveShelfAppIds(source, limit)` without forwarding `sort` / `shelfId` / `sortReverse`, so the resolver fell back to its default (effectively alphabetical for most sources). Forwarding the full sort context + extending the shelf-cache key to include the sort fields makes the promoted shelf honour the user's chosen ordering AND re-resolves automatically when the user edits the sort.

- **Free-games shelf STILL missed currently-free titles even after the v2 fix.** CDP root-causing: Steam's `?specials=1` paginates at ~100 rows per page regardless of `count`, so the v2 store fetcher (one specials hit + one maxprice=free hit + one combined hit) only ever saw the first 100 specials. Additionally `?category2=18` (Free Weekend tag) returns a disjoint set the price-based endpoints don't reach. v3 expands the fetcher to 7 endpoints: specials page 1 (`start=0`), page 2 (`start=100`), page 3 (`start=200`), `maxprice=free`, `specials=1&maxprice=free` (intersection), `category2=18` (Free Weekend tag), and `popular`. Bumped `STORE_KEY` v2 → v3 so existing users re-fetch on next mount instead of waiting on the 6 h TTL of the now-incomplete v2 cache.

- **Store + 100%-discount shelves were rendering empty even when matching games existed on the Steam store.** CDP-validated root cause: free-weekend / free-play-days titles (e.g. `3771740` / `1035110` / `1070580` at time of fix) are CURRENTLY DISCOUNTED 100% but Steam's `?specials=1` query EXCLUDES them, AND `?maxprice=free` only returns permanent F2P. The 3 known free-weekend titles only appeared when both `?specials=1` AND `?maxprice=free` were combined into one query (count 0 from each separate endpoint, count 3 from the combined). Added a 4th endpoint (`?specials=1&maxprice=free`) alongside the existing 3 so the store fetcher actually catches the free-weekend bucket. Plus: bumped store-cache key v1 → v2 (forces existing users to re-fetch instead of waiting on the 6h TTL of the now-incomplete v1 cache), paired the cache with optional `priceHints` (price data when carried by search rows — currently unused since `cc=us` rows only carry `name + logo`, kept as forward-compat for other locales / variants), and added `replayPriceHints` that runs on every getStoreGameIds call so any cached price hints overwrite stale `unpriced:true` entries (relevant when Steam later starts returning price data in rows for some `cc`).

- **Online shelves were hiding games the user doesn't own.** The render-time name-based dedup pulled every non-Steam shortcut AND every cloud-play entry into the "owned name" set unconditionally. The name set now honors the same nested toggles as the appid set, so wishlist items stop matching cloud-play names from cloud-gaming services / Unifideck.
- **Wishlist showed locally-owned non-Steam games whose names differed only by punctuation** (a punctuation-only mismatch). `normalizeTitleForMatch` now lowercases, strips trademark glyphs, and collapses non-alphanumeric runs via Unicode `\p{L}\p{N}` so a title with a colon and the same title without one normalise to the same key. Series suffixes and accented variants stay distinct.
- **Filter-source sort + reverse weren't persisting** (single-key OR multi-key). `filterGroupToFilter` now accepts and propagates `sortReverse`; the resolver prefers `f.sortReverse` on every branch so the editor toggle reaches every filter shelf.
- **Wishlist + store shelves rendered with a multi-key sort that included a price key.** Resolver builds a unified pool (real overviews for local apps, `{ appid }` stubs for remote wishlist items) and runs `applySortToIds` once with a pre-fetched `priceMap`, so the primary discount key orders the whole wishlist instead of only the local subset.
- **Multi-key sort chains with a price key now use the price as a real comparator** (not an alphabetical fallback). `applySortToIds` accepts an optional `priceMap` and threads it through `buildKeyComparator`.
- **Collection picker in the Edit Shelf modal no longer empties after Steam boot.** A 30 s controller refresh + modal-local re-fetch at 500 ms / 2 s / 10 s catches Steam's late `collectionStore` population.
- **QAM action row alignment.** Decky's native `Field` reserves ~half the row width for the empty label slot; pass `childrenLayout='below'` + `childrenContainerWidth='max'` so the action focusable owns the full row.
- **Edit Shelf preview shows the discount + NEW badges**, sized for the smaller preview cards, sitting above the focus indicator, and not clipped at the top. Home shelf badges keep their portal path (focus-ring wins there); only the modal preview uses the inline path. Per-frame badge tracking was removed — badges re-anchor on scroll / resize / focus / blur instead, eliminating the wobble during scale animations.
- **Bundle chunking for `onlineStore`.** Forced into its own chunk so `getWishlistIds` / `getPriceMap` / `getStoreGameIds` resolve at runtime; the previous bundle rewrote the dynamic import to the main bundle which didn't carry those exports.
- **Games added via the library menu's "Add to shelf" now appear in the shelf.** `applyManualOrder` was filtering manualOrder through the resolver's idSet, silently dropping any appid not in the underlying source (which is exactly the menu-add case — the game wasn't in the collection/tab/filter). Hidden ids are dropped from both branches so menu-added games can't bypass hide.

- **Per-shelf hero is no longer the boot bottleneck.** Hoisted the native-hero class scan to a module-level singleton (1 MutationObserver + 1 image scan for all shelves instead of N), rAF-throttled each shelf's subtree MutationObserver, 30 ms slot-swap debounce on rapid d-pad navigation, and skipped the 700 ms hashed-URL refine when `getNativeHeroUrls` already returned the hashed URL. - **Eager-loaded ref callbacks** on both portrait and hero img — when a cached blob URL or HTTP-cache hit means `img.complete && naturalWidth > 0` is true the tick React assigned `src`, the load flag flips synchronously in the same tick. Cached images skip the React render cycle + opacity flash entirely.
- **Initial portrait `src` walks the fallback chain for cache hits.** The chain starts at `/customimages/{appid}p.png` which 404s for most games, so every remount used to pay 1-2 useless local-path 404s + `onError` cycles before reaching the cached CDN URL. - **Image cache no longer churns on `steamloopback.host` URLs.** Those URLs are Steam's local loopback server — serving them as blob URLs adds zero perf benefit, and with 150+ visible cards on a populated home they dominated the 120-entry LRU and constantly evicted the actually-remote CDN URLs (Cloudflare / akamaihd) that DO benefit. `cacheable()` now excludes them.
- **`HOT_CACHE_LIMIT` raised 120 → 320** to fit 150+ visible cards each holding a portrait + hero URL without churn (~38 MB peak memory at saturation).

### Fixed

- **Smart shelf visibility boundary — stale-cache and missed re-render.** Three bugs combined to keep a shelf visible up to 60 min after its window closed: (1) `triggerShelfRefresh()` at the boundary notified `ShelfView` components but did not cause `HomeInject` to re-render, so `isInVisibilityWindow` was never re-evaluated to remove the shelf from the array; (2) the 60-min `resolveSmartShelf` cache could stay valid after the window ended, returning stale game IDs on the next resolve; (3) shelves without explicit `visibleHours` (e.g. `spare_time` relying on its internal `isSpareTimeWindow` check) had no boundary timer at all. Fixed by: adding `visibilityTick` state that increments on each boundary fire (forcing `HomeInject` re-render and re-evaluation of `isInVisibilityWindow`), calling `invalidateSmartShelfCache` for all time-aware shelf IDs before `triggerShelfRefresh`, and using `getModeVisibilityWindows` as fallback when `visibleHours` is absent so modes with internal time checks also get a boundary timer. The effect re-arms via `visibilityTick` in its deps so subsequent boundaries are also caught.
- **Card shrink/distort flicker during fast horizontal navigation (#39).** Steam's native focus rule applies a `1.02×` `scale` transform with a CSS-animated transition when focus enters/leaves a card. The 3 s `discoverNativeCardDimensions` poll could land on the transition tail, measuring an intermediate `1.01×` `getBoundingClientRect()` width on a non-`:focus` / non-`gpfocus` card; that reading then fed `nativeDimsListeners` and briefly shrank every shelf card on the home until the next stable poll. `discoverNativeCardDimensions` now reads `getComputedStyle(...).transform`, parses the matrix, and skips any candidate whose x-scale isn't ~`1` (`Math.abs(sx - 1) > 0.005`). `:focus` / `:hover` / `gpfocus` short-circuits are kept; this layer catches the transition tail those don't.
- **Focus restore on return from a game targets the wrong tree (#38).** `findNavNodeForElement` searched only the `GamepadUI_Full_Root` nav tree, so on SteamOS builds where the home is registered under a different tree id (notably 3.7.x with `root_1_`) the saved `m_lastFocusNode` couldn't be located and Steam's default first-card landing won. The function now iterates `m_ActiveContext.m_rgGamepadNavigationTrees` and walks each tree's root, so `saveFocusTarget` / `tryRestoreFocus` resolve cards regardless of which tree the active context registers them in. Modal `Focusable` wrappers also benefit (relevant to in-modal `focusElement` callers).

### Performance

- **`getAllAppOverviews` — in-flight de-duplication.** When many shelves resolve concurrently before the 10 s cache is populated (typical on cold mount and on resume from sleep), each call used to start its own `GetAllAppOverviews` chain — multiple Steam IPC round-trips per shelf, plus the fallback walks through every Steam window/client. Added an `appOverviewPending` shared promise so the second-and-later concurrent callers `await` the first call's result instead of duplicating the work. The 10 s cache and the post-fetch enrichment / filtering steps are unchanged; this only collapses the in-flight herd.
